# Design Principles

These are the guiding principles for all design and implementation decisions.

## 1. Simplicity over cleverness
Avoid over-engineering. Write code that is easy to read and maintain.

## 2. Traceability
All complex decisions must be documented via Implementation Decision Records (`IR-xxx.md`). 

## 3. Separation of Concerns
Keep components and modules isolated to their specific responsibilities.

## 4. Testability
If it's hard to test, it's designed wrong.
